- The module has three internal layers: (1) `components/ui/` contains 16 presentational components wrapping @base-ui/react primitives, each exposing a consistent API with cva-driven variants and data-slot attributes; (2) `lib/` holds cross-cutting utilities — utils.ts exports the cn() Tailwind merge helper used by every UI component, and api-client.ts provides a typed fetch wrapper with automatic JWT token refresh and error handling; (3) providers.tsx composes TanStack React Query and TooltipProvider into a single app-level provider tree, while middleware.ts implements a minimal Next.js pass-through middleware deferring auth checks to client-side guards.
- Dependency direction flows inward: UI components depend only on lib/utils and @base-ui/react; api-client.ts is standalone; providers.tsx depends on both the tooltip UI component and @tanstack/react-query.
