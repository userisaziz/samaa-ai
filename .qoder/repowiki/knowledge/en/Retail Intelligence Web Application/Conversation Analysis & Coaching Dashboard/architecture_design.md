- **Route layer**: Next.js App Router pages under `(dashboard)` group — `recordings/page.tsx` (list with filtering/pagination), `recordings/[id]/page.tsx` (detail view), and `coaching/page.tsx` (performance analytics).
- **Feature components** (`components/features/`): Four reusable UI modules — `TranscriptViewer` renders speaker-labeled transcript segments with conversation highlighting; `ConversationTimeline` displays color-coded conversation segments on a time axis; `AIInsightsPanel` shows per-conversation AI analysis cards (intent, outcome, objections, scores); `ConversationDrawer` opens a side-sheet with detailed analysis, coaching notes, and filtered transcript.
- **Data flow**: All pages use `@tanstack/react-query` with a shared `api` client to fetch recordings, transcripts, conversations, analyses, and performance metrics. The detail page orchestrates parallel queries (recording, summary, transcript, conversations, per-conversation analyses) and manages `activeConversationId` state to synchronize highlights across timeline, transcript, and insights panels.
- **Boundary**: Feature components are pure presentational units accepting typed props from `@samaa/shared`; they contain no direct API calls except `ConversationDrawer`, which encapsulates its own queries for lazy-loaded analysis data.
